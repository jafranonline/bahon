## 1. Overview

Bahon (বাহন — Bengali for "vehicle" or "carrier") is a free, fully offline-first Progressive Web App for personal vehicle management. It tracks fuel costs, service records, expenses, and maintenance reminders for cars and motorcycles. The app is designed primarily for users in Bangladesh, with full Bangla language support, BDT currency default, and local unit conventions — while remaining fully internationalised for other markets.

The app has no backend, no account required, no ads, and no paid tier. All data is stored locally on the user's device using IndexedDB. Optional cloud backup is delivered through the user's own Google Drive or exported as CSV.

### Design philosophy

- Offline-first: every feature works without internet
- Privacy-first: no data leaves the device unless the user explicitly exports
- Speed: core actions (log fuel, check stats) reachable in under 3 taps
- Simplicity: no VIN entry, no chassis number, no account setup
- Reusable components: every UI element is a self-contained React component with no dependency on platform primitives

### 4.4 Component architecture rules

Every component in `src/components/` must follow these rules:

1. **No platform primitives** — do not use native HTML `<input>`, `<button>`, `<select>` directly in screens. Use the components from `src/components/primitives/` which wrap them.
2. **Props-driven** — all state comes from props or hooks, never internal component state for data.
3. **Single responsibility** — one component, one job.
4. **Typed** — full TypeScript interfaces for all props, no `any`.
5. **Accessible** — all interactive components have ARIA labels, keyboard support, and visible focus states.
6. **Theme-aware** — use CSS custom properties from `tokens.css`, never hardcoded hex values.
7. **Internationalised** — all user-visible strings come from i18n keys, never hardcoded.

## 6. Screens

### 6.1 Home screen (`/`)

**Purpose:** Dashboard — monthly summary, vehicle switcher, quick stats, recent activity.

**Sections (top to bottom):**
1. TopBar — app name, month, settings icon
2. Vehicle selector strip — horizontal scroll pills, one per vehicle + "Add" pill
3. Hero card — monthly total in serif display font, delta vs last month, breakdown row (Fuel / Service / Other)
4. Donut chart card — breakdown visualisation with inline legend
5. 2×2 stat grid — Avg mileage, Distance, Total fuelled, Due services
6. "Quick add" row — 4 action buttons (Fuel, Service, Expense, Reminder)
7. "Recent" section header
8. Last 5 log rows (fuel + service + expense, mixed)
9. Bottom nav

**State:**
- Active vehicle (from vehicleStore)
- Current month logs (from useFuelLogs, useServiceLogs, useExpenses)
- Monthly totals (calculated)

### 6.2 Stats screen (`/stats`)

**Purpose:** Historical analysis across all log types.

**Sections:**
1. TopBar — "Stats", active vehicle name, export button, settings icon
2. Tab bar — Summary / Fuel / Service
3. Summary tab:
   - 2×2 KPI cards: Year to date, Cost per km, Best mileage, Avg monthly
   - Stacked bar chart: monthly fuel + service + other (6 months)
   - Mileage trend line chart (last 10 fill-ups)
   - Recent logs list
4. Fuel tab:
   - Average efficiency trend
   - Fill-up history table
   - Price per litre trend
5. Service tab:
   - Service log list by category
   - Total service cost bar

### 6.3 Log fuel screen (`/log/fuel`)

**Purpose:** Record a fuel fill-up.

**Sections:**
1. TopBar with back arrow
2. Vehicle selector row (current vehicle + change button)
3. Date field
4. Entry mode selector (3-segment):
   - Volume + price/L → auto-calculates total
   - Volume + total → back-calculates price/L
   - Total only → records total without efficiency calculation
5. Volume and price/L inputs (or volume + total, or total only depending on mode)
6. Computed result card (shows total, formula used, efficiency pill if odometers provided)
7. Lock fuel price row — toggle to persist price for next entries
8. Odometer fields (current + previous) — efficiency calculates when both filled
9. Station name (optional)
10. Notes (optional)
11. Save button

**Calculations:**
- `totalCost = volumeLitres × pricePerLitre`
- `pricePerLitre = totalCost / volumeLitres`
- `efficiencyKmL = (currentOdo - prevOdo) / volumeLitres`
- `efficiencyL100km = (volumeLitres / (currentOdo - prevOdo)) × 100`
- `efficiencyMPG = ((currentOdo - prevOdo) × 0.621371) / (volumeLitres × 0.264172)`

**Purpose:** Record a maintenance or service event.

**Fields:** Vehicle, Date, Category (chip selector), Title, Cost, Odometer, Shop name, Notes.

**Purpose:** Add a new vehicle.

**Fields:**
1. Type selector (Car/SUV, Motorcycle, Truck, Bus, Other) — card grid
2. Nickname (free text, required)
3. Brand (free text)
4. Model (free text)
5. Year (number, 4-digit)
6. Current odometer (number, km/mi)
7. Fuel type (chip selector: Octane, Petrol, Diesel, CNG, Electric, Hybrid)

### 6.7 Reminders screen (`/reminders`)

**Purpose:** View and create service/event reminders.

**Sections:**
1. Active reminders list — sorted by urgency (overdue → urgent → upcoming → future)
2. "Add reminder" form:

- Overdue (red): past due date or past due odometer

## 7. Features specification

### 7.1 Fuel logging

- Three entry modes: Volume + price/L, Volume + total, Total only
- Auto-calculation of all derived values
- Efficiency calculated automatically when odometer fields are provided
- Fuel price lock: user can lock current price/L; it pre-fills on next log entry
- Price lock persists across sessions (localStorage)
- Full edit and delete support for any log entry

### 7.2 Service logging

- 20 predefined categories with icons
- Custom category via "Other" + free text title
- Cost required; odometer optional
- Automatically links to active reminder when applicable (marks reminder as completed)

### 7.3 Expense logging

- 9 predefined expense categories
- Amount + category + title + date + notes
- Contributes to monthly "Other" total in dashboard

### 7.4 Reminders

#### One-time reminders
- Trigger: specific date AND/OR odometer reading
- Lead time: configurable days before date (default 3), configurable km before odometer (default 500)
- Status: Pending → Due → Completed / Dismissed

#### Repeat reminders
- Repeat units: Daily, Weekly, Monthly, Yearly, By km
- Configurable interval: "every 3 months", "every 5000 km"
- Auto-advances next due date/odometer when dismissed
- Lead time: same as one-time
- Examples: oil change every 3,000 km, insurance every year, wash every week

#### Reminder notifications
- Web Push API via service worker
- Notification fires at midnight on the lead day
- Tap notification → opens Reminders screen
- Requires permission from user; graceful fallback if denied

### 7.5 Language support

- English (default) and Bangla (বাংলা)
- All UI strings from i18n JSON files
- Bangla uses Hind Siliguri font (covers full Bangla Unicode range)
- Number fields remain in English digits (0–9) for ease of entry
- Date fields use standard HTML date input (locale-aware by browser)
- Language setting persists in localStorage
- Switching language is instant (no reload required)

### 7.6 Theme

- Light mode (default)
- Dark mode
- System (follows OS preference)
- Switching theme is instant
- All colours from CSS custom properties — no hardcoded hex in components

### 7.7 Units

| Setting | Options |
|---------|---------|
| Distance | km (default), mi |
| Volume | L (default), gal |
| Efficiency | km/L (default), L/100km, MPG |

All stored values are always in SI units (km, L). Conversion happens at display time only.

### 7.8 Currency

Supported: BDT (৳, default), USD ($), EUR (€), INR (₹), GBP (£), SAR (ر.س)

All stored values are in the user's selected currency at time of entry. Currency symbol shown beside all amounts.

### 7.9 Data export

- Export as CSV (all logs, all vehicles, all reminders)
- Uses File System Access API with fallback to `<a download>`
- Separate CSV files per entity: vehicles.csv, fuel_logs.csv, service_logs.csv, expenses.csv, reminders.csv
- UTF-8 with BOM for Excel compatibility with Bangla text

### 7.10 Google Drive backup (optional, future)

- User authenticates with Google OAuth2 (PKCE flow, no server)
- App saves compressed JSON to a named file in Drive
- Restore: load JSON, merge with local DB (last-write-wins by updatedAt)

### 7.11 PWA install

- Custom install banner (not browser default)
- iOS: "Add to Home Screen" instruction shown in a bottom sheet
- Android: beforeinstallprompt handled, custom UI button
- Install state persists (don't show again once dismissed)

### 9.2 Bangla numerals

Display uses English numerals (0–9) throughout for:
- All input fields (user entry)
- All computed values
- Chart axes

## 11. Accessibility

- All interactive elements have `aria-label` or visible text label

- Colour contrast ratio ≥ 4.5:1 for all text
